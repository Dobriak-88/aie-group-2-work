## 2. Постановка задачи и контекст

1. **Предметная область и задача**  
   Проект решает задачу **регрессии** – предсказание рыночной цены подержанного автомобиля на основе его характеристик.  
   **Потенциальные пользователи** – площадки по продаже автомобилей (агрегаторы, частные продавцы), которые хотят быстро оценить адекватную цену автомобиля, а также покупатели, желающие проверить справедливость цены.

2. **Формулировка задачи в терминах ML/ИИ**  
   - **Входные данные**: набор характеристик автомобиля: марка, модель, год выпуска, объём двигателя, пробег, количество цилиндров, тип топлива, тип кузова, наличие кожаного салона, тип коробки передач, привод, количество дверей, расположение руля, цвет, количество подушек безопасности.  
   - **Целевая переменная**: логарифм цены автомобиля (в рублях).  
   - **Ограничения/требования**:  
     - Скорость инференса на уровне десятков миллисекунд (для веб-сервиса).  
     - Интерпретируемость не является критической, но желательна (поэтому дополнительно анализировали важность признаков).  
     - Модель должна быть устойчива к редким значениям производителей/моделей (объединяем их в `other`).

3. **Целевые метрики качества**  
   - **RMSE** – основная метрика, так как штрафует за большие отклонения (особенно актуально для дорогих автомобилей).  
   - **MAE** – дополнительная метрика для интерпретируемости (легче объяснить среднюю абсолютную ошибку в логарифмическом масштабе).

## 4. Модели и подходы

Все эксперименты находятся в `notebooks/Test_models.ipynb` и `notebooks/best_model_tune.ipynb`.

1. **Базовые (baseline) модели**  
   - **DummyRegressor** (стратегия `mean`) – RMSE ≈ 0.967, MAE ≈ 0.740.  
   - **RandomForest** (базовый, `n_estimators=100`) – RMSE ≈ 0.462, MAE ≈ 0.266.  
   - **XGBoost** (базовый) – RMSE ≈ 0.645, MAE ≈ 0.447.

2. **Улучшенные модели и эксперименты**  
   - **RandomForest + GridSearchCV** – подбирали `n_estimators`, `max_depth`, `min_samples_split`, `min_samples_leaf`, `max_features`. Лучшие параметры: `max_depth=None`, `max_features=0.5`, `min_samples_leaf=1`, `min_samples_split=2`, `n_estimators=300`.  
     Результат: RMSE = 0.4496, MAE = 0.2593.  
   - **XGBoost + GridSearchCV** – сетка по `n_estimators`, `max_depth`, `learning_rate`, `subsample`, `colsample_bytree`. Лучший результат: RMSE = 0.535, MAE = 0.359 – хуже, чем у RandomForest.  
   - **Анализ важности признаков** (permutation importance) – показал, что `Airbags`, `Manufacturer`, `Prod. year` наиболее значимы, а `Doors` и `Color` – наименее.  
   - **Удаление признаков**:  
     - Удаление `Doors` практически не изменило качество (RMSE 0.4502).  
     - Удаление `Cylinders`, `Color`, `Doors` ухудшило RMSE до 0.4585.  
   - **Итоговая модель** – RandomForest с подобранными гиперпараметрами (сохранена в `artifacts/`).

3. **Нейросетевые модели** – не применялись (табличные данные среднего размера, деревья показали достаточную эффективность).

     - Более быстрое обучение и инференс по сравнению с XGBoost (в рамках данной реализации).

## 6. Архитектура решения и сервис

1. **Архитектура пайплайна**  
   Пайплайн обработки данных включает несколько последовательных этапов. Сначала исходные данные из файла `cars.csv` проходят предобработку с помощью функции `prepare_raw_data`, которая очищает, нормализует и фильтрует признаки. Затем происходит разделение на матрицу признаков `X` и целевую переменную `y` (логарифм цены). Далее применяется препроцессор, состоящий из `StandardScaler` для числовых признаков и `OneHotEncoder` для категориальных. После этого выполняется обучение модели `RandomForestRegressor` с подбором гиперпараметров через `GridSearchCV`. Финальная обученная модель и препроцессор сохраняются в формате joblib в директорию `artifacts/`. При запуске сервиса эти артефакты загружаются и используются для предсказаний.

2. **API и endpoints**  
   Сервис реализован на FastAPI и предоставляет следующие эндпоинты:  
   - `GET /health` – проверка работоспособности сервиса (возвращает статус 200 при наличии загруженной модели).  
   - `POST /predict` – принимает JSON-объект с характеристиками автомобиля (те же признаки, что использовались при обучении), возвращает предсказанную цену в рублях.  
   - `GET /manufacturers` – возвращает список всех производителей, которые встречались в обучающих данных (включая группу `other`).  
   - `GET /models` – возвращает список всех моделей, которые встречались в обучающих данных (включая группу `other`).  
   Для удобства тестирования и документирования доступен интерактивный Swagger UI по адресу `/docs`.

3. **Технологический стек**  
   Проект написан на Python 3.10. Основные библиотеки: pandas, numpy, scikit-learn (для моделей и препроцессинга), xgboost (использовался в экспериментах). Для создания API выбран FastAPI в связке с сервером uvicorn. Контейнеризация реализована с помощью Docker. Для тестирования используется pytest. Все команды для установки зависимостей, запуска обучения и старта сервиса приведены в `README.md`.

## 7. Наблюдаемость, конфигурация и безопасность

1. **Логи и наблюдаемость**  
   - Логирование организовано через модуль `src.utils.logger`. При импорте модуля автоматически настраивается логгер `car_price` с выводом в консоль (`sys.stdout`) и в файл `src/logs/car_price.log` (кодировка UTF-8).  
   - Формат логов: `время - имя логгера - уровень - сообщение`.  
   - В сервисе логируются входящие запросы к `/predict` (уровень INFO), ошибки валидации (ERROR), исключения при загрузке модели или предсказании (ERROR).  
   - Health-check эндпоинт `/health` позволяет мониторить доступность сервиса (200 при успешной загрузке модели, 503 при её отсутствии или повреждении).  
   - Лог-файл не ротируется автоматически – это потенциальное улучшение для производственной среды.

2. **Конфигурации**  
   - В директории `configs/` хранятся два JSON-файла с метаинформацией о моделях:  
     - `best_model_meta.json` – параметры и метрики финальной модели RandomForest после тюнинга гиперпараметров.  
     - `best_model_meta_test.json` – результаты первичного сравнения моделей (базовый RandomForest, XGBoost, DummyRegressor).  
   - В корне проекта присутствует файл `.env.example` с шаблоном переменных окружения (`DATA_PATH`, `ARTIFACTS_PATH`, `LOGS_PATH`, `RANDOM_STATE`). В текущей реализации переменные окружения не используются – пути и параметры жёстко заданы в коде. Это сделано для упрощения запуска, но может быть доработано в будущем для гибкости.  
   - Пути к данным (`data/cars.csv`), артефактам (`artifacts/`) и конфигам вынесены в переменные в модулях, что позволяет легко их изменять при необходимости.

3. **Безопасность**  
   - Репозиторий не содержит чувствительных или конфиденциальных данных (паролей, токенов, ключей API, персональной информации).  
   - Используемый датасет `cars.csv` является открытым (источник – Kaggle), не содержит персональных данных или коммерческой тайны.  
   - Артефакты (модель, препроцессор) не содержат исполняемого кода и не представляют угрозы при распространении.  
   - Файл `.env.example` присутствует как шаблон, но в текущей реализации переменные окружения не используются, поэтому риск утечки отсутствует.

## 8. Ограничения и дальнейшая работа

**Текущие ограничения:**

- Модель обучена только на предоставленном датасете (19 237 строк) и не обновляется автоматически при появлении новых данных.
- Не поддерживаются все возможные производители и модели (редкие заменяются на `'other'`), что может снижать точность для экзотических автомобилей.
- API работает синхронно, отсутствует кэширование результатов.
- При первом запуске API требуется предварительно обученная модель (необходимо выполнить `src.train` до старта сервиса).

**Направления для развития:**

- Добавить ансамбль из RandomForest и XGBoost для повышения точности предсказаний.
- продумать как собирать новые данные и добавить автопереобучение при добавление новых данных.
