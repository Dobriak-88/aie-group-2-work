## 2. Структура проекта

Проект организован в следующей структуре:

- `requirements.txt` – зависимости проекта (библиотеки Python, необходимые для запуска).
- `report.md` – отчёт по проекту (постановка задачи, данные, эксперименты, результаты).
- `self-checklist.md` – чеклист самопроверки проекта перед сдачей.
- `notebooks/` – экспериментальные ноутбуки:
  - EDA, прототипы моделей, отдельные эксперименты.
- `src/` – основной код проекта:
  - модули для работы с данными;
  - обучение моделей;
  - запуск сервиса/скриптов.
- `data/` – демонстрационные/учебные данные:
  - **только открытые, синтетические или обезличенные данные;**
  - при необходимости – небольшие выборки для быстрого запуска.
- `configs/` – конфигурационные файлы:
  - `.yaml`, `.json`, `.env.example` и т.п.
- `tests/` – тесты (юнит-тесты, простые проверки).
- `artifacts/` – сохранённые модели, отчёты, результаты экспериментов.

## 3. Требования и установка

### 3.1. Требования

- Python `>= 3.10`
- Docker

### 3.2. Установка окружения

```bash
# Перейти в папку проекта
cd project

# Создать виртуальное окружение (рекомендуется)
python -m venv .venv

# Активировать окружение:
# Windows:
.venv\Scripts\activate
# Linux / macOS:
source .venv/bin/activate

# Установить зависимости
pip install --upgrade pip
pip install -r requirements.txt

# Собрать Docker-образ (для работы с итоговым продуктом)
docker build -t car-price-predictor .
```

---

## 4. Как запустить проект

### 4.1. Запуск обучения модели

```bash
cd project
source .venv/bin/activate      # при необходимости
python -m src.train
```

### 4.2. Запуск сервиса (API)

```bash
cd project
source .venv/bin/activate      # при необходимости
python -m src.run_api
```

Сервис запускается на порту `8000`.

**Эндпоинты:**  
- `GET /health` – проверка работоспособности  
- `POST /predict` – предсказание цены автомобиля  
- `GET /manufacturers` – список допустимых производителей  
- `GET /models` – список допустимых моделей  

**Проверка работоспособности:**  
Откройте в браузере `http://localhost:8000/docs` – интерактивная документация Swagger.

**Запуск через Docker**

```bash
cd project
docker build -t car-price-predictor .
docker run --rm -v "$(pwd)/artifacts:/app/artifacts" car-price-predictor src.train
docker run -p 8000:8000 car-price-predictor src.run_api
```

## 5. Данные

- **Источник:** открытый датасет с Kaggle [Car Prices Dataset](https://www.kaggle.com/datasets/sidharth178/car-prices-dataset) (Sidharth Kumar Mohanty).
- **Размер:** 19 237 строк, 18 столбцов (включая целевую переменную `Price`).
- **Состав:** характеристики автомобиля: производитель, модель, год выпуска, тип кузова, объём двигателя, пробег, количество цилиндров, тип топлива, коробка передач, привод, цвет, количество подушек безопасности и др.
- **Используемые поля:** все, кроме столбца `ID`. Обучение и оценка выполняются на случайном разбиении тренировочных данных (80/20).
- **Расположение в репозитории:** `src/data/cars.csv` (файл уже включён в репозиторий, отдельно скачивать не требуется).
- **Предобработка:** удаление дубликатов, приведение регистра, замена редких производителей/моделей на `'other'`, фильтрация выбросов (по цене, году выпуска, пробегу, объёму двигателя).

## 8. Ограничения и дальнейшая работа

**Текущие ограничения:**

- Модель обучена только на предоставленном датасете (19 237 строк) и не обновляется автоматически при появлении новых данных.
- Не поддерживаются все возможные производители и модели (редкие заменяются на `'other'`), что может снижать точность для экзотических автомобилей.
- API работает синхронно, отсутствует кэширование результатов.
- При первом запуске API требуется предварительно обученная модель (необходимо выполнить `src.train` до старта сервиса).

**Направления для развития:**

- Добавить ансамбль из RandomForest и XGBoost для повышения точности предсказаний.
- продумать как собирать новые данные и добавить автопереобучение при добавление новых данных.
