# Отчет по выполнению практической работы
## Прогнозирование временного ряда с использованием классических методов и нейронных сетей

### 1. Введение
В рамках данной работы решалась задача прогнозирования временного ряда на основе предоставленного датасета. Целью являлось построение и сравнение нескольких моделей: базовых (наивная, скользящее среднее) и более сложных, включая линейную регрессию с ручным созданием признаков и рекуррентную нейронную сеть (GRU). Особое внимание уделялось корректному временному разбиению данных и воспроизводимости результатов.

### 2. Постановка задачи
Задача – одномерное прогнозирование на один шаг вперед (горизонт 1) для временного ряда `target` из датасета `S12-hw-dataset.csv`. Необходимо обучить модели на исторических данных и оценить их качество на тестовом периоде. Метрики оценки: MAE, RMSE, MAPE.

### 3. Описание данных
Датасет содержит 4320 записей за период с 1 января 2025 года по 29 июня 2025 года. Временной шаг – предположительно 1 час (4320 записей / 180 дней = 24 точки в день). Разбиение выполнено строго по времени:
- **Train**: первые 70% (3024 записи) – с 2025-01-01 по 2025-05-06.
- **Validation**: следующие 15% (648 записей) – с 2025-05-07 по 2025-06-02.
- **Test**: оставшиеся 15% (648 записей) – с 2025-06-03 по 2025-06-29.

График исходного ряда и разбиения представлен в отчёте (рисунок `series_split.png`). Пропуски в данных отсутствуют.

### 4. Методология
#### 4.1. Признаки для классических моделей
Для моделей baseline (naive, moving-average) и Ridge были сформированы следующие признаки:
- Календарные: день недели, месяц, синус/косинус дня недели и месяца.
- Лаги: 1, 7, 14, 28.
- Rolling-статистики: среднее и стандартное отклонение за 7 и 28 предыдущих значений (со сдвигом, чтобы избежать утечки данных).
- Все признаки масштабировались с помощью `StandardScaler`.

#### 4.2. Baseline модели
- **B1: naive-last** – прогноз равен последнему известному значению (лаг 1).
- **B2: moving-average** – прогноз равен скользящему среднему за последние 7 наблюдений.
- **B3: ridge-lag-features** – линейная регрессия с L2-регуляризацией (Ridge) на всех построенных признаках.

#### 4.3. Нейросетевая модель (GRU)
Для учета временной зависимости использовалась рекуррентная нейросеть GRU с архитектурой:
- Вход: окно из 28 последних значений (скалированных по train).
- Один слой GRU с 64 скрытыми нейронами.
- Полносвязный слой на выходе.
- Функция потерь: MSE, оптимизатор: Adam (lr=1e-3).
- Ранняя остановка (patience=10) по loss на валидации.
- Масштабирование выполнено отдельно по train, с применением тех же параметров к val и test.

#### 4.4. Метрики
- **MAE** (Mean Absolute Error)
- **RMSE** (Root Mean Squared Error)
- **MAPE** (Mean Absolute Percentage Error)

### 5. Эксперименты и результаты
Все модели были обучены и оценены на тестовом наборе. Результаты сведены в таблицу ниже:

|-----------------------|----------|----------|----------|
| B3: ridge-lag-features| 4.969621 | 6.516360 | 3.256090 |
| B1: naive-last        | 6.366645 | 8.107879 | 4.168898 |
| R1: GRU               | 7.337949 | 9.301221 | 4.694508 |
| B2: moving-average    |12.750535 |15.260929 | 8.562792 |

График сравнения MAE представлен на рисунке `baselines_compare.png`. Также для лучшей модели (Ridge) и GRU построены графики прогнозов на тестовом периоде (рисунок `best_forecast_test.png`). Кривые обучения GRU показаны на `gru_learning_curves.png`.

### 6. Обсуждение результатов
- **Ridge** показала наилучшие результаты по всем метрикам. Это объясняется тем, что она использовала богатый набор признаков (лаги, скользящие статистики, календарные) и хорошо улавливает линейные зависимости в данных.
- **Naive-last** (простое повторение последнего значения) оказалась вполне конкурентоспособной, что характерно для рядов с высокой автокорреляцией.
- **Moving-average** дала ожидаемо худший результат, так как сглаживает резкие изменения.
- **GRU** не смогла превзойти Ridge, несмотря на способность моделировать нелинейные зависимости. Возможные причины: недостаточный объём данных (всего ~3000 обучающих точек), неоптимальные гиперпараметры (размер окна, архитектура), а также отсутствие дополнительных признаков (модель использует только историю целевой переменной). Возможно, использование более глубокой архитектуры или других типов рекуррентных слоёв (LSTM) могло бы улучшить результат.

### 7. Выводы
В ходе работы реализовано несколько подходов к прогнозированию временного ряда. Лучший результат достигнут с помощью линейной модели Ridge на ручных признаках. GRU показала средний результат, уступив простым статистическим методам. Полученные результаты подчёркивают важность тщательного построения признаков и показывают, что для данного набора данных классические методы могут быть более эффективны, чем сложные нейросетевые архитектуры.
